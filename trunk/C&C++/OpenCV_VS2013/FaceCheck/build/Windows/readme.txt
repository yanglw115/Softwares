一、projectTest（基于64位）
    1、目前因为dlib库的原因，项目只能使用VS2017打开并运行及调试（之前好像是可以使用VS2013调试的，不知从何时起无法使用了，dlib头文件检测C++11特性不支持）；
    2、glog目前也无法正常在VS2017上面使用，运行时报内存访问出错（之前也是没有问题的，可能是系统环境有冲突）；
    3、最近使用VS2017可以进行正常调试，编译也使用64位系统进行（怀疑VS2013不能使用的原因也可能是后来包括dlib、glog等库编译并替换成了64位，只是没有时间去验证，也不是必要的）；

二、projectJni（基于64位）
    1、用于编译出JNI库；
    2、需要注意的是，编译的时候依赖系统安装的JDK或JRE，因为需要包含jni.h等相关头文件，可以放在系统环境变量或者放在VS工程配置当中（建议放在VS工程配置当中，添加相关头文件到附加包含目录）；
    3、直接使用release编译选项；

三、package目录里面，是JAVA在windows平台的测试程序（所有的编译及运行基本64位）。
    1、生成的JNI动态库文件可以放在libs里面，主要是用于SVN备份与管理；    2、运行时所依赖的libs库文件，取决于系统环境变量的配置，所以要成功运行JAVA测试程序，还需要为JNI动态库配置环境变量或者放在JDK的库目录（建议配置环境变量）。目前建议将生成的JNI动态库文件拷贝一份到D:\CureFaceParser\libs下面，并将此路径添加到系统环境变量；
    3、执行.bat自动运行脚本前，需要配置好JAVA的环境变量；
